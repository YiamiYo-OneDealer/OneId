### Self-contained JWT (claim-in-token) architecture
Decision: embed Permission array, Dimensional Attributes, and License status directly in the JWT. OneDealer v2 validates locally — no runtime introspection calls to OneId per request. Risk: large claim sets may exceed safe HTTP header size limits (4–6KB practical budget). POC must validate worst-case token size. Fallback: reference tokens + introspection endpoint. Logged to addendum.

### License state embedding: status not live count
Live seat counts are mutable and stale immediately. Token embeds `license.status` (active/exceeded), evaluated at issuance. Users already authenticated remain active until token expiry even if Tenant later exceeds limit.

### SAML2 library TBD
OpenIddict has no native SAML2 support. Library selection (Sustainsys.Saml2 vs ITfoxtec.Identity.Saml2) is an open question for the POC. Options captured in addendum.

### Hybrid JWT + enriched introspection chosen
Decision: OneId issues signed JWTs containing Role names. Server-side `jti` records enable revocation. Introspection endpoint validates token (including revocation check) and enriches response with Permissions, Dimensional Attributes, and License state from the database. OneDealer v2 validates JWT locally for role checks; calls introspection (cached) for fine-grained claims. Rationale: avoids JWT size risk while giving OneDealer v2 fast local role checks without a network call. Role-change propagation solved by server-side revocation — effective staleness window = introspection cache TTL, not JWT TTL.

### Server-side token revocation on role change
When a User's Roles change, OneId deletes their active token records by `jti`. Next introspection call returns `active: false`; OneDealer v2 triggers silent refresh; new JWT reflects updated Roles. Only access tokens revoked; refresh tokens untouched unless action is a security event.

### Azure AD added to IDP federation scope (POC)
Azure AD (Microsoft Entra ID) added alongside Okta as a supported upstream IDP. Same pattern: OIDC relying party, identity mapped by email/UPN, claims not propagated.

### Introspection cache TTL fixed at 5 minutes
5-minute cache TTL is the explicit propagation delay for all claim changes (role revocations, permission updates, dimensional attribute changes). This is the accepted design tradeoff — not a defect.

### Suspension = immediate token revocation for all Tenant Users
Suspending a Tenant deletes all active `jti` records for all Users in that Tenant. Next introspection call returns `active:false` across the board.

### Active User (seat-count) = non-deactivated account
For seat-count licensing: active seat = any User account not explicitly deactivated by a Tenant Admin. Definition is model-specific; concurrent and usage-based models will define differently.

### Auto-provisioning deferred from POC
Federated users (Okta, Azure AD) must be pre-provisioned in OneId. If no matching User exists, auth fails. Future feature: configurable claim-based auto-provisioning rules (email domain, incoming claim matching → default Tenant/Group/Role assignment).

### Dimensional Attribute semantics confirmed
OR logic within an axis (User sees data matching ANY of their assigned values per axis); AND logic across axes (data must match on ALL constrained axes). Users can hold multiple values per axis. Attributes assigned directly per User, not via Roles/Groups.
